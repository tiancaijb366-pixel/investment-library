# 5 Debugging Prompts That Save Me Hours

## 1. The Rubber Duck
```
You are a senior debugging engineer. I'll show you my code and the error.
Walk through what the code should do, then what actually happens.
Identify the exact line where reality diverges from expectation.
```

## 2. The Root Cause
```
This error occurs intermittently. List every possible cause,
ranked by probability. For each cause, suggest a way to confirm or rule it out.
```

## 3. The Minimal Reproduction
```
Help me create a minimal reproduction of this bug.
What parts of the codebase can I strip away while keeping the bug alive?
```

## 4. The Log Analysis
```
Here are my error logs. Parse them into:
- Timeline of events
- First point of failure
- Cascading failures
- Suggested fix priority
```

## 5. The Regression Finder
```
This code used to work. Compare the current implementation
with the expected behavior. What changed?
```
